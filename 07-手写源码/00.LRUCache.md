# LRUCache

## 简介

LRU（Least Recently Used）缓存是一种常用的缓存淘汰策略：当缓存容量达到上限时，淘汰最久未使用的数据。本实现利用 JavaScript 的 `Map` 数据结构，其迭代顺序为插入顺序，因此 `Map` 中第一个元素就是最久未使用的元素。

## 执行流程

```mermaid
flowchart TD
    A[get(key)] --> B{cache 中有 key?}
    B -->|否| C[返回 -1]
    B -->|是| D[删除旧记录]
    D --> E[重新插入 key-value]
    E --> F[返回 value]
    
    G[put(key, value)] --> H{cache 中有 key?}
    H -->|是| I[删除旧记录]
    H -->|否| J[插入 key-value]
    I --> J
    J --> K{cache size > capacity?}
    K -->|是| L[删除 Map 第一个元素]
    K -->|否| M[结束]
```

## 代码实现

```javascript
//ES6
class LRUCache {
  constructor(capacity) {
    this.cache = new Map();
    this.capacity = capacity;
  }

  get(key) {
    if (!this.cache.has(key)) return -1;

    const v = this.cache.get(key);
    this.cache.delete(key);
    this.cache.set(key, v);
    return this.cache.get(key);
  };

  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key);
    }
    this.cache.set(key, value);
    if (this.cache.size > this.capacity) {
      this.cache.delete(this.cache.keys().next().value);
    }
  };
}

//ES5
var LRUCache = function (capacity) {
  this.capacity = capacity;
  this.map = new Map();
};

LRUCache.prototype.get = function (key) {
  let value = this.map.get(key)
  if (value === undefined) {
    return -1;
  }
  this.map.delete(key);
  this.map.set(key, value);
  return value;
};

LRUCache.prototype.put = function (key, value) {
  if (this.map.has(key)) {
    this.map.delete(key);
  }
  this.map.set(key, value);
  if (this.map.size > this.capacity) {
    this.map.delete(this.map.keys().next().value);
  }
};
```

## 逐行解析

### ES6 版本（class）

- **`constructor(capacity)`**：初始化 `cache` 为空的 `Map` 对象，并设置容量上限 `capacity`。
- **`get(key)`**：如果 key 不存在返回 -1。如果存在，先删除该 key 再重新插入（将其移到 Map 末尾，表示最近使用），最后返回值。
- **`put(key, value)`**：如果 key 已存在则先删除。插入 key-value 对。如果插入后 size 超过容量，通过 `this.cache.keys().next().value` 获取 Map 的第一个 key（最久未使用）并删除。

### ES5 版本（构造函数）

- 逻辑与 ES6 版本完全相同，使用构造函数和原型方法。
- 区别在于使用 `var` 声明和使用 `prototype` 扩展方法。

## 复杂度分析

- **时间复杂度**：get O(1)，put O(1) — Map 的 get、set、delete 操作均为 O(1)
- **空间复杂度**：O(capacity) — 最多存储 capacity 个键值对
