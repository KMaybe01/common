# 实现一个版本号处理函数

将版本号数组从小到大排序，注意每个版本号的段数可能不同，比较时逐段比较。

function sortVersion(list) {
  return list.sort((a, b) => {
    let aa = a.split('.')
    let bb = b.split('.')
    let length = aa.length > bb.length ? aa.length : bb.length
    for (var i = 0; i < length; i++) {
      let x = aa[i] || 0;
      let y = bb[i] || 0;
      if (x - y !== 0) return x - y;
    }
  });
}

## 逐段解析

### 排序逻辑
- 使用 `Array.sort` 方法，传入自定义比较函数
- 比较函数接收两个版本号 a 和 b

### 分段比较
- 将版本号按 `.` 分割为数字段数组 `aa` 和 `bb`
- 取两个数组长度的较大值作为循环次数
- 逐段比较：`aa[i] || 0` 和 `bb[i] || 0`
- 如果某段不存在则默认为 0，这样 `"1.5"` 和 `"1.45.0"` 比较时，`"1.5"` 的第三段为 0，`"1.45.0"` 的第三段也为 0，但第二段 `5 < 45`，所以 `"1.5" < "1.45.0"`
- 这正是题目要求的：注意 `'1.45'` 比 `'1.5'` 大

### 返回值
- 如果 `x - y !== 0`，返回差值作为排序依据
- 如果所有段都相等，返回 0

## 复杂度分析
- **时间复杂度**：O(n log n * m)，n 为数组长度，m 为版本号平均段数
- **空间复杂度**：O(m)，分割数组的临时空间
- **核心要点**：逐段比较、缺失段补零、字符串数字会自动隐式转为数字比较
