# 使用 setTimeout 实现 setInterval

## 简介

使用递归 `setTimeout` 模拟 `setInterval` 功能，确保上一个回调执行完毕后才会触发下一个定时器，避免 `setInterval` 可能出现的回调堆积问题。

## 执行流程

```mermaid
flowchart TD
    A[调用 mySetInterval(fn, timeout)] --> B[创建 timer 控制器 flag=true]
    B --> C[定义 interval 递归函数]
    C --> D[调用 setTimeout(interval, timeout)]
    D --> E[到达 timeout 时间]
    E --> F{timer.flag 为 true?}
    F -->|否| G[停止执行]
    F -->|是| H[执行 fn]
    H --> I[递归调用 setTimeout(interval, timeout)]
    I --> E
    G --> J[返回 timer 控制器]
    J --> K[结束]
```

## 代码实现

```javascript
function mySetInterval(fn, timeout) {
    var timer = {
        flag: true
    };
    function interval() {
        if (timer.flag) {
            fn();
            setTimeout(interval, timeout);
        }
    }
    setTimeout(interval, timeout);
    return timer;
}
```

## 逐行解析

1. **第 8 行**: 定义 `mySetInterval` 函数，接收回调 `fn` 和时间间隔 `timeout`。
2. **第 10-12 行**: 创建 `timer` 控制器对象，包含 `flag` 属性控制定时器是否继续运行。
3. **第 13-18 行**: 定义 `interval` 递归函数，检查 `timer.flag` 为 `true` 时执行 `fn()`，然后递归调用 `setTimeout(interval, timeout)` 开启下一个定时器。
4. **第 20 行**: 首次启动定时器。
5. **第 22 行**: 返回 `timer` 控制器，外部可通过设置 `timer.flag = false` 停止定时器。

## 复杂度分析

- **时间复杂度**: O(1)，每次调用执行常数量操作，递归次数由外部控制。
- **空间复杂度**: O(1)，仅保留 `timer` 控制器对象。
