# 实现 Request 并发控制器

> 控制最大并发请求数量，每次最多发起 max 个请求，完成一个后再启动下一个，全部完成后调用回调。

## Mermaid 流程图

```mermaid
flowchart TD
    A[输入 urls 数组, max 并发数] --> B[初始化 i=0, requestsQueue=[], results=[]]
    B --> C[调用 handleRequest urls[0] 启动]
    C --> D[fetch url]
    D --> E[results.push res]
    E --> F{全部完成? len === urlCount?}
    F -->|是| G[调用 callback results]
    F -->|否| H{还有待发请求?}
    H -->|是| I[shift 移除已完成请求]
    I --> J[handleRequest urls[++i] 启动下一个]
    H -->|否| K[等待其他请求完成]

    subgraph 并发控制
        L[当前 queue 长度 < max?]
        L -->|是| M[继续启动下一个请求]
        L -->|否| N[等待已有请求完成]
    end
```

//写一个并发request函数，能够控制并发请求数量，并请求结束后调用回调函数，请求直接使用fetch发送。
function handleFetchQueue(urls, max, callback) {
  const urlCount = urls.length;
  const requestsQueue = [];
  const results = [];
  let i = 0;
  const handleRequest = (url) => {
    const req = fetch(url).then(res => {
      console.log('当前并发： ' + requestsQueue);
      const len = results.push(res);
      if (len < urlCount && i + 1 < urlCount) {
        requestsQueue.shift();
        handleRequest(urls[++i])
      } else if (len === urlCount) {
        'function' === typeof callback && callback(results)
      }
    }).catch(e => {
      results.push(e)
    });
    if (requestsQueue.push(req) < max) {
      handleRequest(urls[++i])
    }
  };
  handleRequest(urls[i])
}

## 逐行解析

### handleFetchQueue 主函数
- **`urlCount`**：请求总数。
- **`requestsQueue`**：正在执行中的请求队列，用于控制并发数。
- **`results`**：收集所有请求的结果。
- **`i`**：当前已启动的请求索引。
- **`handleRequest(url)`**：核心递归函数，处理单个请求。
  1. 发起 `fetch(url)`，请求完成后将结果 `push` 到 `results`。
  2. 如果还有未发起的请求（`len < urlCount && i + 1 < urlCount`），从 `requestsQueue` 中移除完成的请求，启动下一个。
  3. 如果所有请求完成（`len === urlCount`），调用 `callback`。
  4. 在启动前，通过 `requestsQueue.push(req) < max` 判断当前并发数是否达到上限，未达上限则继续启动下一个请求。
- **初始调用**：`handleRequest(urls[i])` 从第一个请求开始。

### 并发控制逻辑
- 每次 `requestsQueue.push(req)` 后检查长度，如果 `< max` 就继续启动新请求。
- 当一个请求完成时，`shift` 移除它，然后启动下一个，保证 `requestsQueue` 中最多有 `max` 个请求。

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(n) | n 为请求总数，每个请求处理为 O(1) |
| 空间复杂度 | O(n) | results 数组存储所有结果，requestsQueue 存储当前并发请求 |
| 并发上限 | max | 通过 `requestsQueue.length < max` 控制 |
