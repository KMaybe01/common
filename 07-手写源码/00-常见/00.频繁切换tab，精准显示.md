# 防抖函数与频繁切换Tab的精准请求控制

> 利用防抖（Debounce）机制，在频繁切换 Tab 时只执行最后一次请求，避免无效的网络请求。

## 源代码

```javascript
/**
 * 函数防抖，一定时间内连续触发事件只执行一次
 * @param {*} func 需要防抖的函数
 * @param {*} delay 防抖延迟
 * @param {*} immediate 是否立即执行，为true表示连续触发时立即执行，即执行第一次，为false表示连续触发后delay ms后执行一次
 */
let debounce = function (func, delay = 100, immediate = false) {
    let timeoutId, last, context, args, result

    function later() {
        const interval = Date.now() - last
        if (interval < delay && interval >= 0) {
            timeoutId = setTimeout(later, delay - interval)
        } else {
            timeoutId = null
            if (!immediate) {
                result = func.apply(context, args)
                context = args = null
            }
        }
    }

    return function () {
        context = this
        args = arguments
        last = Date.now()

        if (immediate && !timeoutId) {
            result = func.apply(context, args)
            context = args = null
        }

        if (!timeoutId) {
            timeoutId = setTimeout(later, delay)
        }

        return result
    }
}


let flag = false // 标志位，表示当前是否正在请求数据
let xhr = null

let request = (i) => {
    if (flag) {
        clearTimeout(xhr)
        console.log(`取消第${i - 1}次请求`)
    }
    flag = true
    console.log(`开始第${i}次请求`)
    xhr = setTimeout(() => {
        console.log(`请求${i}响应成功`)
        flag = false
    }, Math.random() * 200)
}

let fetchData = debounce(request, 50) // 防抖

// 模拟连续触发的请求
let count = 1
let getData = () => {
    setTimeout(() => {
        fetchData(count)
        count++
        if (count < 11) {
            getData()
        }
    }, Math.random() * 200)
}
getData()

```

## 逐行解析

### debounce 函数
- **`let debounce = function (func, delay = 100, immediate = false)`**：定义防抖函数，接受三个参数：需要防抖的函数、延迟时间（默认100ms）、是否立即执行。
- **`timeoutId, last, context, args, result`**：闭包中保存的变量，分别用于存储定时器ID、上次触发时间、this上下文、参数列表、执行结果。
- **`later()`**：内部函数，用于在延迟后检查是否应该执行。计算当前时间与上次触发时间的差值 `interval`，如果 `interval < delay` 说明又有新触发，重新计时；否则执行函数。
- **返回的函数**：每次事件触发时，更新 `last` 为当前时间。如果 `immediate` 为 true 且无定时器（第一次触发），立即执行。然后设置新的定时器。
- **`if (immediate && !timeoutId)`**：第一次触发时立即执行，后续触发只重置定时器。
- **`context = args = null`**：执行后解除引用，避免内存泄漏。

### 请求控制部分
- **`flag`**：标志位，表示当前是否正在等待请求响应。
- **`xhr`**：存储当前请求的定时器ID，用于取消。
- **`request(i)`**：模拟请求函数。如果 `flag` 为 true，说明上次请求未完成，`clearTimeout` 取消它。然后设置新的定时器模拟请求。
- **`fetchData = debounce(request, 50)`**：对 request 函数应用防抖，50ms 防抖延迟。
- **`getData()`**：模拟连续触发，随机间隔（0-200ms）调用 `fetchData`，共触发10次。

## 复杂度分析

| 维度 | 复杂度 | 说明 |
|------|--------|------|
| 时间复杂度 | O(1) | 防抖函数每次触发只执行常数次操作 |
| 空间复杂度 | O(1) | 仅使用固定数量的闭包变量 |
| 适用场景 | 频繁触发的事件 | Tab切换、搜索输入、窗口resize等 |
