# 手写一个观察者、发布订阅模式

## 简介

观察者模式和发布订阅模式非常相似，核心都在于**注册**和**触发**。区别在于：
- **观察者模式**：目标（Subject）直接维护观察者（Observer）列表，状态变化时直接通知所有观察者。（如 Vue2 响应式）
- **发布订阅模式**：通过中间层（PubSub）解耦发布者和订阅者，发布者触发中间层的 publish，中间层通知所有订阅者。

## 执行流程

```mermaid
flowchart TD
    subgraph 观察者模式
        A[Subject 状态变化] --> B[遍历 Observer 列表]
        B --> C[调用 observer.update]
    end
    
    subgraph 发布订阅模式
        D[Publisher 发布] --> E[publisher.publish(pubsub)]
        E --> F[pubsub 遍历 subscribes]
        F --> G[调用 subscribe.update]
    end
```

## 代码实现

```javascript
//观察者 - 简易版
var subject = {
    observers: [],
    notify() {
        this.observers.forEach(observer => {
            observer.update()
        })
    },
    attach(observer) {
        this.observers.push(observer)
    }
}
var observer = {
    update() {
        console.log('观察者updated')
    }
}
subject.attach(observer)
subject.notify()

// 观察者模式: class 版
class ObserverList {
    constructor() {
        this.observerList = [];
    }
    add(observer) {
        this.observerList.push(observer);
        return this;
    }
    remove(observer) {
        this.observerList = this.observerList.filter(ob => ob !== observer);
        return this;
    }
    get(index) {
        return this.observerList[index];
    }
    count() {
        return this.observerList.length;
    }
}

class Subject {
    observers = new ObserverList;
    add(observer) {
        this.observers.add(observer);
    }
    remove(observer) {
        this.observers.remove(observer);
    }
    notify(...params) {
        for (let i = 0; i < this.observers.count(); i++) {
            let item = this.observers.get(i);
            item.update(...params);
        }
    }
}

class Observer {
    update(message) {
        console.log('消息接收Observer！', message);
    }
}
class Demo {
    update(message) {
        console.log('消息接收Demo！', message);
    }
}

let sub = new Subject;
sub.add(new Observer);
sub.add(new Observer);
sub.add(new Demo);
setTimeout(() => {
    sub.notify('你好~！');
}, 1000);

// 发布订阅模式
var publisher = {
    publish(pubsub) {
        pubsub.publish()
    }
}
var pubsub = {
    subscribes: [],
    publish() {
        this.subscribes.forEach(subscribe => {
            subscribe.update();
        })
    },
    subscribe(sub) {
        this.subscribes.push(sub)
    }
}
var subscribe = {
    update() {
        console.log('订阅者update')
    },
    subscribe(pubsub) {
        pubsub.subscribe(this);
    }
}
subscribe.subscribe(pubsub)
publisher.publish(pubsub)
```

## 逐行解析

### 观察者模式 - 简易版
- **`subject`**：被观察的目标对象，维护一个 `observers` 数组。
- **`attach(observer)`**：注册观察者，将其添加到列表中。
- **`notify()`**：通知所有观察者，遍历列表调用每个观察者的 `update` 方法。
- **`observer`**：观察者对象，提供 `update` 方法供目标调用。

### 观察者模式 - class 版
- **`ObserverList`**：封装观察者列表的管理操作（增、删、查、计数）。
- **`Subject`**：目标类，持有 `ObserverList` 实例。提供 `add`、`remove`、`notify` 方法。
- **`Observer`**：观察者类，实现 `update` 方法。
- 可以灵活添加不同类型的观察者（如 `Demo` 类），只要实现 `update` 方法即可。

### 发布订阅模式
- **`publisher`**：发布者，只负责调用中间层 `pubsub` 的 `publish` 方法，不直接操作订阅者。
- **`pubsub`**：中间调度层，维护 `subscribes` 列表，提供 `subscribe`（订阅）和 `publish`（发布）方法。
- **`subscribe`**：订阅者，实现 `update` 方法。
- 发布者和订阅者完全解耦，通过中间层通信。

## 复杂度分析

- **观察者模式**：
  - 注册 O(1)，通知 O(n) — n 为观察者数量
  - 空间复杂度 O(n)
- **发布订阅模式**：
  - 订阅 O(1)，发布 O(n) — n 为订阅者数量
  - 空间复杂度 O(n)
