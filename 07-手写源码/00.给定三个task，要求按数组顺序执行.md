# 给定三个 task，要求按数组顺序执行

三个异步任务（定时器），需要按照数组顺序依次执行，前一个完成后才能执行下一个。

## 流程图

```mermaid
flowchart TD
    A[func([task1, task2, task3])] --> B{args 为空?}
    B -- 否 --> C[取出第一个任务<br/>args.shift]
    C --> D[new Promise 包装]
    D --> E[执行任务<br/>传入 resolve]
    E --> F[任务完成调用 callback<br/>即 resolve]
    F --> G[then 递归调用 func]
    G --> B
    B -- 是 --> H[结束]
```

## 代码与解析

```javascript
const task1 = (callback) => setTimeout(() => {
    console.log(1);
    callback();
}, 500);
const task2 = (callback) => setTimeout(() => {
    console.log(2);
    callback();
}, 200);
const task3 = (callback) => setTimeout(() => {
    console.log(3);
    callback();
}, 300);
// 同步实现func([task1, task2, task3])
const func = args => {
    if (args.length > 0) new Promise(resolve => args.shift()(resolve)).then(() => func(args))
}
func([task1, task2, task3])
```

- **任务结构**：每个 task 接受一个 `callback` 参数，异步操作完成后调用 `callback`
- **核心逻辑**：
  - `args.shift()` 从数组中取出第一个任务
  - 将 `resolve` 作为 `callback` 传入任务
  - 任务完成后调用 `callback()` 即 `resolve()`，触发 `.then()`
  - `.then()` 中递归调用 `func(args)` 处理下一个任务
- 即使 `task2` 的定时器时间（200ms）比 `task1`（500ms）短，也会等到 `task1` 完成后再执行
- 输出顺序：1 → 2 → 3

## 复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) — n 为任务数量 |
| 空间复杂度 | O(n) — 递归调用栈（可改为迭代优化） |
