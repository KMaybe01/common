# 手动实现 EventEmitter（发布订阅模式）

## 简介

EventEmitter 是发布订阅模式的典型实现，提供事件注册（on）、事件触发（emit）、事件移除（off）和一次性监听（once）四个核心方法。本实现分别使用 ES5 构造函数和 ES6 class 两种方式实现。

## 执行流程

```mermaid
flowchart TD
    subgraph 注册 on
        A[on(type, callback)] --> B{_events[type] 存在?}
        B -->|否| C[_events[type] = [callback]]
        B -->|是| D[_events[type].push(callback)]
    end

    subgraph 触发 emit
        E[emit(type, ...args)] --> F{_events[type] 存在且非空?}
        F -->|是| G[遍历执行所有 callback]
    end

    subgraph 移除 off
        H[off(type, callback)] --> I{_events[type] 存在?}
        I -->|是| J[filter 过滤掉指定 callback]
    end

    subgraph 一次性 once
        K[once(type, callback)] --> L[创建包装函数 foo]
        L --> M[foo 执行后自动调用 off 移除]
        M --> N[通过 on 注册 foo]
    end
```

## 代码实现

```javascript
//ES5
function MyEvent() {
  this._events = Object.create(null)
}

MyEvent.prototype.on = function (type, callback) {
  if (this._events[type]) {
    this._events[type].push(callback)
  } else {
    this._events[type] = [callback]
  }
}

MyEvent.prototype.off = function (type, callback) {
  if (this._events && this._events[type]) {
    this._events[type] = this._events[type].filter((item) => {
      return item !== callback && item.link !== callback
    })
  }
}

MyEvent.prototype.once = function (type, callback) {
  let foo = function (...args) {
    callback.call(this, ...args)
    this.off(type, foo)
  }
  this.on(type, foo)
}

MyEvent.prototype.emit = function (type, ...args) {
  if (this._events && this._events[type].length) {
    this._events[type].forEach((callback) => {
      callback.call(this, ...args)
    })
  }
}

let ev = new MyEvent()
let fn = function (...data) {
  console.log('事件1执行了', data)
}

ev.once('事件1', fn)
ev.emit('事件1', 'once-es5')
ev.emit('事件1', 'once-es5-2')
// 事件1执行了 [ 'once-es5' ]

ev.on('事件1:on', fn)
ev.emit('事件1:on', 'on-es5')
ev.off('事件1:on', fn)
ev.emit('事件1:on', 'on-es5-2')
// 事件1执行了 [ 'on-es5' ]

//ES6
class MyEvent2 {
  constructor() {
    this._events = Object.create(null)
  }
  on(type, callback) {
    if (this._events[type]) {
      this._events[type].push(callback)
    } else {
      this._events[type] = [callback]
    }
  }

  emit(type, ...args) {
    if (this._events && this._events[type].length) {
      this._events[type].forEach((callback) => {
        callback.call(this, ...args)
      })
    }
  }

  off(type, callback) {
    if (this._events && this._events[type]) {
      this._events[type] = this._events[type].filter((item) => {
        return item !== callback && item.link !== callback
      })
    }
  }

  once(type, callback) {
    let foo = function (...args) {
      callback.call(this, ...args)
      this.off(type, foo)
    }
    this.on(type, foo)
  }
}
```

## 逐行解析

- **`_events = Object.create(null)`**：创建一个没有原型的空对象，用于存储事件名到回调函数列表的映射。这样避免了原型链上属性的干扰。
- **`on(type, callback)`**：如果该事件类型已存在，将回调 push 到数组中；否则创建一个新数组。
- **`emit(type, ...args)`**：使用 `...args` 收集参数，遍历执行该事件类型对应的所有回调函数，通过 `call` 绑定 `this`。
- **`off(type, callback)`**：使用 `filter` 过滤掉需要移除的回调函数。`item.link !== callback` 用于处理 `once` 包装的情况。
- **`once(type, callback)`**：创建一个包装函数 `foo`，在 `foo` 中执行原始 `callback` 后立即调用 `off` 移除自己。然后将 `foo` 通过 `on` 注册。
- **ES6 class 版本**：逻辑与 ES5 版本完全相同，使用 class 语法更简洁。

## 复杂度分析

- **on**：O(1) — 直接 push 或赋值
- **emit**：O(n) — n 为该事件注册的回调数量
- **off**：O(n) — filter 需要遍历所有回调
- **once**：O(1) — 包装后调用 on
- **空间复杂度**：O(m × n) — m 为事件类型数，n 为每个事件的平均回调数
