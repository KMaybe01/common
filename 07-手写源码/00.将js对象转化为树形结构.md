# 将JS对象转化为树形结构

实现将扁平化的数组数据（包含id和pid）转换为树形结构，这是前端处理组织架构、菜单树等场景的常见需求。

## 流程图

```mermaid
graph TD
    A[输入: 扁平数组] --> B[创建 id→item 映射表]
    B --> C[遍历数组]
    C --> D{parent = map[item.pid]}
    D -->|parent 存在| E[将当前节点加入 parent.children]
    D -->|parent 不存在| F[当前节点为根节点, 加入 result]
    E --> G[返回树形结构]
    F --> G
    
    H[递归方式] --> H1[取第一个元素为根]
    H1 --> H2[递归查找子节点]
    H2 --> H3[构建嵌套 children]
```

function jsonToTree(data) {
    // 初始化结果数组，并判断输入数据的格式
    let result = []
    if (!Array.isArray(data)) {
        return result
    }
    // 使用map，将当前对象的id与当前对象对应存储起来
    /*
     {
       '1': { id: 1, pid: 0, name: 'body' },
       '2': { id: 2, pid: 1, name: 'title' },
       '3': { id: 3, pid: 2, name: 'div' }
     }
    */
    let map = {};

    data.forEach(item => {
        map[item.id] = item;
    });
    data.forEach(item => {
        let parent = map[item.pid];
        if (parent) {
            (parent.children || (parent.children = [])).push(item);
        } else {
            result.push(item);
        }
    });
    return result;
}

function arrayToTree(array) {
    let root = array[0]
    array.shift()
    let tree = {
        id: root.id,
        val: root.val,
        children: array.length > 0 ? toTree(root.id, array) : []
    }
    return tree;
}

function toTree(parenId, array) {
    let children = []
    let len = array.length
    for (let i = 0; i < len; i++) {
        let node = array[i]
        if (node.parentId === parenId) {
            children.push({
                id: node.id,
                val: node.val,
                children: toTree(node.id, array)
            })
        }
    }
    return children
}

## 逐段解析

### jsonToTree - 映射表法（推荐）
- **建立映射表**：第一次遍历，以每个元素的 id 为 key，元素本身为 value，存入 map 对象
- **构建树关系**：第二次遍历，通过 map 查找当前元素的 parent（即 pid 对应的元素）。如果存在 parent，将当前元素加入 parent.children 数组；如果没有对应的 parent，说明是根节点，加入 result
- **优点**：时间复杂度 O(n)，只需遍历两次，无需递归

### arrayToTree - 递归法
- 取数组第一个元素为根节点
- 递归查找 parentId 等于当前节点 id 的元素，构建 children 数组
- **缺点**：每次递归都要遍历整个数组，效率较低

### jsonToTree2 - 映射表法（第二种数据格式）
- 与 jsonToTree 原理相同，只是 pid 字段名不同（使用 parentId）
- 支持任意深度的嵌套

## 复杂度分析
- **映射表法**：时间复杂度 O(n)，空间复杂度 O(n)
- **递归法**：时间复杂度 O(n²)，空间复杂度 O(n)
- **推荐**：映射表法，一次遍历建立映射，二次遍历构建树，效率最高
