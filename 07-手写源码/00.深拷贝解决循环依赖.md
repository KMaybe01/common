# 深拷贝解决循环依赖

普通递归深拷贝无法处理循环引用（如 `obj.e = obj`），会导致栈溢出。本文使用 `WeakMap` 缓存已拷贝对象来解决循环依赖。

## 流程图

```mermaid
flowchart TD
    A[输入 source] --> B{isObject?}
    B -- 否 --> C[直接返回 source]
    B -- 是 --> D{hash.has(source)?}
    D -- 是 --> E[返回 hash 中缓存的 target]
    D -- 否 --> F[创建空 target<br/>数组或对象]
    F --> G[hash.set(source, target)]
    G --> H[for...in 遍历 source]
    H --> I{hasOwnProperty?}
    I -- 是 --> J{isObject(source[key])?}
    J -- 是 --> K[递归 cloneDeep<br/>传入 hash]
    K --> L[target[key] = 结果]
    J -- 否 --> M[target[key] = source[key]]
    I -- 否 --> H
    L --> N[返回 target]
    M --> N
```

- 普通递归深拷贝遇到 `obj.e === obj`（循环引用）时会无限递归，导致栈溢出

### 解决方案：WeakMap 缓存

```javascript
function isObject(obj) {
    return (typeof obj === "object" || typeof obj === "function") && obj !== null;
}

function cloneDeep(source, hash = new WeakMap()) {
    if (!isObject(source)) return source;
    if (hash.has(source)) return hash.get(source); // 新增代码，查哈希表

    var target = Array.isArray(source) ? [] : {};
    hash.set(source, target); // 新增代码，哈希表设值

    for (var key in source) {
        if (Object.prototype.hasOwnProperty.call(source, key)) {
            if (isObject(source[key])) {
                target[key] = cloneDeep(source[key], hash); // 新增代码，传入哈希表
            } else {
                target[key] = source[key];
            }
        }
    }
    return target;
}

var obj = {
    a: 1,
    b: 2,
    c: [1, 2, 3],
    d: {
        aa: 1,
        bb: 2
    },
};
obj.e = obj;
console.log("obj", obj);
const objCopy2 = cloneDeep(obj);
console.log(objCopy2);
```

- **`isObject`**：判断一个值是否为对象或函数（排除 `null`）
- **`WeakMap` 缓存**：
  - 每次拷贝一个对象前，先存入以 `source` 为 key、`target` 为 value 的映射
  - 递归中遇到已存在的 `source` 直接返回缓存的 `target`，打破循环
- **`WeakMap` 优势**：键为弱引用，不影响垃圾回收，适合这类临时缓存场景
- 执行结果：`objCopy2.e === objCopy2` 为 `true`，正确保留了循环引用结构

## 复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) — n 为对象中所有节点数，每个节点只访问一次 |
| 空间复杂度 | O(n) — WeakMap 存储所有节点映射 + 递归调用栈深度 |
