# 手写 bind 函数

## 简介

`bind` 函数用于创建一个新的函数，当被调用时将其 `this` 绑定到指定的上下文，并可以预设部分参数。手动实现 `bind` 需要考虑：判断调用者是否为函数、支持参数柯里化（预设参数）、以及处理当新函数作为构造函数被调用时的特殊 this 指向。

## 执行流程

```mermaid
flowchart TD
    A[fn.myBind(context, ...args)] --> B{判断 this 是否为函数}
    B -->|否| C[抛出 TypeError]
    B -->|是| D[保存 fn = this]
    D --> E[保存预设参数 args]
    E --> F[返回代理函数 Fn]
    F --> G[调用 fn.apply]
    G --> H{this instanceof Fn?}
    H -->|是| I[传入 this 作为上下文]
    H -->|否| J[传入 context 作为上下文]
    I --> K[合并预设参数和新参数]
    J --> K
    K --> L[返回执行结果]
```

## 代码实现

```javascript
Function.prototype.myBind = function (context) {
  if (typeof this !== "function") {
    throw new TypeError("Error");
  }
  let args = [...arguments].slice(1),
    fn = this;
  return function Fn() {
    return fn.apply(
      this instanceof Fn ? this : context,
      args.concat(...arguments)
    );
  };
};

const obj = {
  value: 1,
}

function fn() {
  console.log(this.value, ...arguments); // 1, 2
}

fn.myBind(obj, 2)();
```

## 逐行解析

- **`typeof this !== "function"`**：判断调用 `myBind` 的对象是否为函数，防止在非函数上调用。
- **`let args = [...arguments].slice(1)`**：获取除 `context` 外的其他预设参数（柯里化）。
- **`let fn = this`**：保存当前函数引用，以便在返回的代理函数中调用。
- **`return function Fn() { ... }`**：返回一个新的代理函数。
- **`this instanceof Fn`**：判断当前函数是否通过 `new` 调用（即作为构造函数）。如果是，`this` 指向新创建的对象，此时应该使用 `this` 而不是传入的 `context`。
- **`args.concat(...arguments)`**：将预设参数与新传入的参数合并。
- **`fn.apply(context, mergedArgs)`**：使用 `apply` 绑定 this 并传入合并后的参数执行。

## 复杂度分析

- **时间复杂度**：O(1) — 创建新函数并执行
- **空间复杂度**：O(n) — n 为参数个数，需要保存预设参数和合并参数数组
