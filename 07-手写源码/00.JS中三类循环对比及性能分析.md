# JS 中三类循环对比及性能分析

## 简介

JavaScript 中有三类主要循环：`for`、`for...in`、`for...of`。它们各有优劣：
- **for 循环**：自己控制循环过程，性能最优，配合 `let` 声明时不会创建全局未释放变量。
- **for...in 循环**：迭代所有可枚举属性（包括原型链上的），性能最差，且有遍历顺序、无法遍历 Symbol 等问题。
- **for...of 循环**：基于迭代器（Iterator）规范，只能遍历实现了 `Symbol.iterator` 的数据结构。

## 逐行解析

### for 循环
- **`for (let i = 0; i < arr.length; i++)`**：基于 `let` 声明时，每次迭代都会创建一个新的块级作用域，不会产生全局未释放变量，性能最优。
- **while 循环**：与 for 循环性能相近，适用于不确定循环次数的场景。

### 手动实现 forEach
- **`myForEach`**：通过 `for` 循环遍历数组，将每个元素、索引和原数组通过 `fn.call` 传递给回调函数。
- 使用 `call` 显式指定 `this` 指向。

### for...in 循环的问题
- **遍历顺序以数字优先**：数字属性名按升序排列，字符串属性按创建顺序排列。
- **无法遍历 Symbol 属性**：`for...in` 不会枚举 `Symbol` 类型的属性。
- **遍历到原型链**：`for...in` 会遍历对象原型链上所有可枚举属性（如 `Object.prototype.fn`）。
- **解决方案**：使用 `Object.keys()` + `Object.getOwnPropertySymbols()` 组合。

### for...of 循环和迭代器
- **迭代器规范**：对象需要实现 `Symbol.iterator` 方法，该方法返回一个包含 `next` 方法的迭代器对象。
- `next()` 每次返回 `{ value: 当前值, done: 是否结束 }`。
- **类数组对象**：默认不具备迭代器规范，会抛出 `TypeError`。可以通过 `obj[Symbol.iterator] = Array.prototype[Symbol.iterator]` 借用数组的迭代器。

## 复杂度分析

- **for 循环**：时间复杂度 O(n)，空间复杂度 O(1)
- **for...in 循环**：时间复杂度 O(n + m) — n 为自身属性，m 为原型链属性，空间复杂度 O(1)
- **for...of 循环**：时间复杂度 O(n)，空间复杂度 O(1)
- **性能排序**：for ≈ while > for...of > forEach > for...in
