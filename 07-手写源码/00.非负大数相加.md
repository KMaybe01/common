# 非负大数相加

当数字超过 JavaScript 的 `Number.MAX_SAFE_INTEGER`（9007199254740991）时，无法精确表示。本文实现字符串形式的大数加法，模拟竖式计算。

## 流程图

```mermaid
flowchart TD
    A[输入字符串 s, t] --> B[从右向左逐位相加]
    B --> C[取当前位 s[i] + t[j] + 进位]
    C --> D[当前位 = 和 % 10]
    D --> E[进位 = 和 // 10]
    E --> F{还有未处理位<br/>或进位不为0?}
    F -- 是 --> B
    F -- 否 --> G[结果数组翻转后 join]
```

## 代码与解析

```javascript
// 思路： 模拟加法运算，但是需要用'0'补齐长度，对于整数，向前补0。
let a = "0";
let b = "0";
// 1. 先找出最大的长度的数
// 2. 给较小的数填充向前填充0

function solve(s, t) {
  let i = s.length - 1;
  let j = t.length - 1;
  let add = 0;
  let res = [];
  while (i >= 0 || j >= 0 || add !== 0) {
    let p = i >= 0 ? s[i] - 0 : 0;
    let q = j >= 0 ? t[j] - 0 : 0;
    let sum = p + q + add;
    let cur = sum % 10;
    res.push(cur);
    add = Math.floor(sum / 10);
    j--;
    i--;
  }
  return res.reverse().join("");
}

console.log(solve(a, b));
```

- **逐位相加**：从两个字符串的最右边（个位）开始遍历
- **补 0 处理**：较短的字符串高位视为 0（通过三元运算符处理）
- **进位处理**：`sum % 10` 为当前位，`Math.floor(sum / 10)` 为进位
- **最终翻转**：结果数组是从低位到高位存储的，需要 `reverse` 后再 `join`

## 复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(max(n, m)) — 只需遍历较长字符串的每一位 |
| 空间复杂度 | O(max(n, m)) — 结果数组长度与最长字符串一致 |
