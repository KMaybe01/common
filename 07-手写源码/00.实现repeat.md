# 实现 repeat 方法

实现一个函数，让某个函数可以重复执行指定次数，每次执行之间有固定的时间间隔。

## 流程图

```mermaid

    A[repeat(func, times, wait)] --> B[方法一: setTimeout递归]
    A --> C[方法二: Promise + async/await]
    A --> D[方法三: for-of + Promise]
    A --> E[方法四: setInterval]
    
    B --> B1[内部定义递归函数]
    B --> B2[setTimeout 延迟后执行]
    B --> B3[times--, 继续递归或停止]
    
    C --> C1[返回 async 函数]
    C --> C2[while 循环 await Promise]
    C --> C3[Promise resolve 后执行下一次]
    
    D --> D1[Array.from 创建长度数组]
    D --> D2[for-of + await 依次执行]
    
    E --> E1[setInterval 定时执行]
    E --> E2[count++, 达到次数 clearInterval]
```

//方法一
function repeat1(func, times, wait) {
    const oThis = this;
    let timer = null;

    const rFuntion = (msg) => {
        timer = setTimeout(() => {
            func.call(oThis, msg);
            times--;
            if (times > 0) {
                rFuntion(msg);
            } else {
                clearTimeout(timer);
            }
        }, wait);
    };
    return (msg) => {
        if (timer) clearTimeout(timer);
        rFuntion(msg);
    };
} //实现打印4次，每次间隔 3000ms

function repeat2(func, times, wait) {
    const oThis = this;

        return new Promise(function (resolve) {
            timer = setTimeout(() => {
                func.call(oThis, msg);
                resolve();

            await rFuntion(msg);
            times--;
        }
        clearTimeout(timer);
    };

## 逐段解析

### 方法一：setTimeout 递归
- 内部定义 `rFuntion` 递归函数
- `setTimeout` 延迟 `wait` 毫秒后执行 `func`
- 执行后 `times--`，如果 `times > 0` 继续递归调用
- 返回的函数中先清除上一次的 timer，然后开始执行

### 方法二：Promise + async/await
- `rFuntion` 返回 Promise，setTimeout 完成后 resolve
- 返回的 async 函数中使用 `while` 循环 `await rFuntion(msg)`
- 每次等待 Promise resolve 后再执行下一次，实现间隔效果

### 方法三：for-of + Promise
- 使用 `Array.from({length: times})` 创建指定长度的数组
- `for...of` 配合 `await` 依次执行
- 逻辑更清晰，结构更简洁

### 方法四：setInterval
- 使用 `setInterval` 定时执行
- `count` 累计执行次数，达到 `times` 时 `clearInterval` 清除定时器
- **注意**：setInterval 的执行间隔是从前一次执行开始算，可能累积误差

### 附录：字符串重复（已注释）
- 两种方式：`Array.join` 和递归 concat
- 属于字符串操作，与函数重复调用不同

## 复杂度分析
- **时间复杂度**：O(times)，执行 times 次函数调用
- **空间复杂度**：O(1)
- **推荐**：Promise + async/await 方式，代码更可读，控制更精确
