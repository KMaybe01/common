# JS 数据类型判断

## 简介

JavaScript 中有多种数据类型检测方式：`typeof`、`instanceof`、`constructor` 以及 `Object.prototype.toString.call()`。其中 `Object.prototype.toString.call()` 是最标准、最准确的检测方法。本实现通过封装 `getType` 函数，统一处理基本类型和引用类型的类型判断。

## 执行流程

```mermaid
flowchart TD
    A[getType(value)] --> B{typeof value === 'object'}
    B -->|是| C[Object.prototype.toString.call]
    C --> D[截取 [object Xxx] 中的 Xxx]
    D --> E[转为小写并首字母大写]
    E --> F[返回类型名]
    B -->|否| G{value === null}
    G -->|是| H[返回 'Null']
    G -->|否| I[typeof value]
    I --> J[首字母大写处理]
    J --> F
```

## 代码实现

```javascript
const getType = (value) => {
  if (typeof value === "object") {
    return changeData(Object.prototype.toString.call(value).slice(8, -1).toLowerCase());
  } else if (value === null) {
     return changeData(value);
  } else {
    return changeData(typeof value);
  }
}

const changeData = (value) => {
  return value[0].toUpperCase() + value.slice(1);
};

console.log("========getType==========");
console.log(getType(Symbol())); //Symbol
console.log(getType(null)); //Null
console.log(getType([])); //Array
console.log(getType({})); //Object
console.log(getType(12)); //Number
console.log(getType(undefined)) //Undefined

let obj2 = {
  name: "珠峰培训",
};
console.log(obj2.toString()); // [object Object]
console.log(obj2.valueOf()); // { name: '珠峰培训' }
```

## 逐行解析

- **`typeof value === "object"`**：判断是否为引用类型（包括对象、数组、正则、日期等），`typeof null` 也返回 `"object"`，因此需要单独处理。
- **`Object.prototype.toString.call(value)`**：标准检测方法。对任何类型调用都会返回 `"[object Xxx]"` 格式的字符串，如 `"[object Array]"`、`"[object Object]"`。
- **`.slice(8, -1)`**：截取 `"[object Array]"` 中的 `"Array"` 部分。从索引 8 开始到倒数第一个字符结束。
- **`.toLowerCase()`**：转为全小写，方便统一处理。
- **`value === null`**：单独处理 `null`，因为 `typeof null` 返回 `"object"` 是历史遗留 BUG。
- **`changeData`**：将字符串首字母大写，返回标准类型名。
- **`typeof value`**：处理基本类型（Number、String、Boolean、Undefined、Symbol）和函数类型。

## 复杂度分析

- **时间复杂度**：O(1) — 固定操作，不随输入规模增长
- **空间复杂度**：O(1) — 仅使用常数额外空间
