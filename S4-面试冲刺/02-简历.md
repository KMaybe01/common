## 🚀 项目实践

### 项目一：5GC 网元自动化测试平台
**技术栈**：React 18 + UmiJS + SSE + Kubernetes + Go

> **背景**：5G 核心网网元（SMF / UPF）测试流程复杂，传统 UI 编码方式无法支撑测试用例频繁变动的业务需求，测试过程中的日志排障极其耗时。

- **动态表单引擎** —— 基于 JSON Schema 设计配置驱动引擎，实现联动校验与动态字段渲染，支撑非前端人员快速落地测试场景
- **智能诊断流** —— 利用 SSE 构建实时日志推送，结合前端正则算法实现异常状态秒级高亮，问题定位效率提升 **50%**
- **云原生运维** —— 实现 Kubernetes YAML 在线编辑与下发，将前端作为云原生网元的管理入口，支撑日均 **200+** 自动化测试任务

---

### 项目二：企业级综合网络管理系统
**技术栈**：Angular 20 + RxJS + WebSocket + OpenLayers + ECharts

> **背景**：面对海量基站设备，运维团队需进行实时全息监控。原系统在海量告警并发与 GIS 地图拓扑渲染上存在显著性能瓶颈。

- **高并发流控** —— 利用 RxJS 高阶操作符处理 WebSocket 消息流，在 **1000+ QPS** 压力下实现告警的平滑聚合与渲染，保障监控看板流畅运行
- **GIS 高性能渲染** —— 通过 OpenLayers 空间聚合算法攻克万级点位渲染难题，支持多维过滤与动态实时刷新，满足大规模拓扑展示需求
- **安全认证体系** —— 落地 RBAC 动态权限与双 Token 无感刷新机制，在高频运维环境下实现 **99.9%** 的会话稳定性

---

### 项目三：网元运维与数据管理系统
**技术栈**：Angular 20 + PrimeNG + Tailwind CSS + Prometheus + Grafana

> **背景**：网元溯源场景涉及百万行级日志文件预览，常规预览方式常导致浏览器内存溢出（OOM），严重影响一线运维排障效率。

- **大文件性能攻坚** —— 针对 **25MB+ 百万行日志**，采用流式分片读取（ReadableStream）+ 虚拟列表方案，从底层解决内存瓶颈，实现毫秒级加载
- **可观测性建设** —— 集成 Prometheus + Grafana，实现网元健康指标与前端监控的深度融合，通过数据驱动排障，减少无效的现场运维成本
