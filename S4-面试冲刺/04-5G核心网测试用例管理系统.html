项目概述 ​

一、项目背景 ​

在5G核心网（5GC）的开发与测试过程中，测试工程师需要对大量的网络功能（NF）进行配置和验证。传统的手工配置方式效率低下、容易出错，且难以管理复杂的测试场景。本项目旨在构建一个可视化的测试用例管理系统，通过Web界面实现对5G核心网测试环境的统一管理和自动化测试。

2.1 动态表单DSL渲染引擎 ⭐⭐⭐ ​

位置: src/pages/TCList/TCModel/components/RenderForm.tsx (362行)

难点分析 ​

这是一个后端驱动的表单DSL系统，前端需要将后端返回的JSON Schema递归解析并渲染为完整的Ant Design表单。

核心挑战:

  1. 无限层级嵌套: tabs → tab → card → form → leaf fields，支持任意深度
  2. 6种叶子字段类型: string、int、float、select、intArray、floatArray
  3. 动态路径映射: 每个表单项的name需要根据嵌套层级动态构建
  4. 数组类型支持: ProFormList实现动态增删，每项包装在可折叠ProCard中

2.2 树形数据操作引擎 ⭐⭐⭐ ​

位置: src/pages/TCList/TCModel/components/edit.ts (209行)

难点分析 ​

事件树是一个可编辑的嵌套结构，需要支持：

  1. 递归增删改查: 任意层级节点的CRUD操作
  2. Key唯一性保证: 复制/添加时重新生成所有子孙节点的Key
  3. 操作权限控制: 不同类型节点（select/map）有不同的操作权限
  4. 递归校验: 遍历所有节点收集校验失败的Key

核心算法 ​

Key重新生成算法:

typescript
export const updateKeysForAddOrCopy = (
  data: treeItem[], 
  type: 'ADD' | 'COPY', 
  parentKey = ''
) => {
  data?.forEach((item: treeItem) => {
    if (item?.key) {
      // 保留原始Key的最后一段作为标识
      const currentKeys = item?.key?.split('-');
      item.key = newKey + currentKeys.pop();
      
      // ADD模式清空值，COPY模式保留值
      if (type === 'ADD') delete item?.value;
    }
    
    // 子节点使用随机数生成唯一Key
    if (item?.children && Array.isArray(item?.children)) {
      newKey = parentKey 
        ? `${parentKey}-${(Math.random() * 1000000).toFixed(0)}` 
        : item?.key;
      updateKeysForAddOrCopy(item?.children, type, newKey);
    }
  });
  return [{ ...data[0], key: parentKey }];
};

export const generateItemToKey = (data: treeItem[], map = new Map()) => {
  for (const item of data) {
    if (item.type === 'select') {
      // select类型: 子节点可复制，多于1个时可删除
      item.children?.forEach((ch) => map.set(ch.key + 'copy', true));
      if (item.children?.length > 1) {
        item.children.forEach((ch) => map.set(ch.key + 'delete', true));
      }
    } else if (item.type === 'map') {
      // map类型: 子节点可复制、可删除、可添加map
      item.children?.forEach((ch) => {
        map.set(ch.key + 'map', true);
        map.set(ch.key + 'copy', true);
      });
      if (item.children?.length > 1) {
        item.children.forEach((ch) => map.set(ch.key + 'delete', true));
      }

递归校验算法:

typescript
export const validateAllKeys = (data: treeItem[]) => {
  const typeLists = ['text', 'float', 'select', 'digit', 'bool', 
                     'digitArray', 'floatArray', 'textArray'];
  let keys: string[] = [];
  
  data?.forEach((item) => {
    if (!item?.children && item.require && typeLists.includes(item?.type)) {
      // 校验空值
      if (!isNotEmpty(item?.value)) {
        keys.push(item?.key);
      } else {
        // 校验格式规则
        if (item?.type === 'digitArray') {
          const expr = /^\d+(?:(,\d+)+)?$/.test(item?.value);
          if (!expr) keys.push(item?.key);
        } else if (item?.type === 'floatArray') {
          const expr = /^(-?\d+(\.\d+)?)(,-?\d+(\.\d+)?)*$/.test(item?.value);
          if (!expr) keys.push(item?.key);
        }
      }
    }
    if (item?.children) {
      keys = keys.concat(validateAllKeys(item?.children));
    }
  });
  return keys;
};

useEffect(() => {
  if (!logModalOpen || !fgcgen) return;

  // 1. 创建EventSource连接（浏览器原生SSE客户端）
  const eventSource = new EventSource(
    `${getApiPrefix()}/Fgcgen/logs?namespace=${fgcgen.namespace}&pod=${fgcgen.name}`,
  );

  // 2. 监听服务端推送的消息

    setLogData((prev) => prev + event.data);
  };

  // 3. 连接错误时仅记录日志，让浏览器自动重连
  eventSource.onerror = () => {
    console.error('SSE error, will auto-reconnect');
  };

  // 4. 组件卸载或依赖变化时清理
  return () => {
    eventSource.close();
    setLogData('');
  };
}, [logModalOpen, fgcgen]);

// 建造者：分步构建复杂对象
class FormConfigBuilder {
  private formConfiguration = {};
  private pathMapToData: string[] = [];

  // 步骤1：构建NF配置映射
  buildNFMapping(formJson: any): this {
    if (formJson?.children?.length) {
      formJson.children.forEach((item: any) => {
        this.formConfiguration = {
          ...this.formConfiguration,
          [item.key]: item.children[0],
        };
      });
    }
    return this;
  }

  // 步骤2：递归提取路径映射
  buildPathMapping(): this {
    ['amf', 'udm', 'pcf', 'chf', 'nrf', 'upf', 'sgw'].forEach((nf) => {
      this.traverseNFConfig(this.formConfiguration[nf]);
    });
    return this;
  }

  // 步骤3：递归遍历
  private traverseNFConfig(obj: any): void {
    if (Array.isArray(obj)) {
      obj.forEach((item) => this.traverseNFConfig(item));
    } else if (typeof obj === 'object' && obj !== null) {
      if (obj?.controlType === 'form' && obj?.path !== '') {
        Object.keys(obj.data || {}).forEach((key) => {
          this.pathMapToData.push(`${obj.path}.${key}`);
        });
      }

Q11: React 19的并发模式是什么？项目中如何应用？ ​

答:

并发模式核心:

  • React可以同时准备多个版本的UI
  • 高优先级更新（用户输入）可以打断低优先级更新（数据加载）

// 1. startTransition - 标记非紧急更新
const handleSave = (record: treeItem, value: any) => {

    item.key === data.key ? { ...data, tree: recordList } : item
  );
  // 再异步更新state（非紧急）

优势:

  • 编辑表格时不会卡顿
  • 模式切换更流畅
  • 大数据量更新不阻塞交互
