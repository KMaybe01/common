## 项目概述

### 一、项目背景

在5G核心网（5GC）的开发与测试过程中，测试工程师需要对大量的**网络功能（NF）**进行配置和验证。传统的手工配置方式效率低下、容易出错，且难以管理复杂的测试场景。本项目旨在构建一个**可视化的测试用例管理系统**，通过Web界面实现对5G核心网测试环境的统一管理和自动化测试。

### 二、核心定位

| 属性 | 说明 |
|------|------|
| **项目名称** | TCS Management (Test Case System Management) |
| **产品定位** | 5G核心网SMF（Session Management Function）测试工具管理界面 |
| **目标用户** | 5G网络测试工程师、开发工程师 |
| **部署环境** | Docker容器化 → OpenShift/K8s集群（内网部署） |
| **访问方式** | 浏览器访问，Hash路由模式 |

## 二、技术难点深度剖析

### 2.1 动态表单DSL渲染引擎 ⭐⭐⭐

**位置**: `src/pages/TCList/TCModel/components/RenderForm.tsx` (362行)

#### 难点分析

这是一个**后端驱动的表单DSL系统**，前端需要将后端返回的JSON Schema递归解析并渲染为完整的Ant Design表单。

**核心挑战**:
1. **无限层级嵌套**: `tabs → tab → card → form → leaf fields`，支持任意深度
2. **6种叶子字段类型**: string、int、float、select、intArray、floatArray
3. **动态路径映射**: 每个表单项的`name`需要根据嵌套层级动态构建
4. **数组类型支持**: ProFormList实现动态增删，每项包装在可折叠ProCard中

#### 设计方案

```typescript
// 三函数递归架构
renderForm(item, path, currentTab)     // 顶层分发器: tabs/card/form
  └─→ renderFormItem(data, path, currentTab)  // 处理Object/Array嵌套
        └─→ renderData(data, path, currentTab, type)  // 渲染叶子节点
              └─→ renderChildren(children, path, currentTab)  // 递归子项
```

**类型到控件映射表**:

| controlType | 渲染组件 | 特殊处理 |
|-------------|----------|----------|
| `string` | ProFormText | hasFeedback实时校验 |
| `int` | ProFormDigit | precision: 0, min/max安全整数 |
| `float` | ProFormDigit | precision: 10, 无步进器 |
| `select` | ProFormSelect | 支持boolean选项转换 |
| `intArray` | ProFormText | 正则校验 + transform数组转换 |
| `floatArray` | ProFormText | 正则校验 + transform数组转换 |
| `tabs` | Tabs + ProCard | 递归渲染tab子项 |
| `card` | ProCard | 可折叠，递归渲染children |
| `form` | 递归renderFormItem | 支持Object/Array两种模式 |

**数组类型特殊处理**:
```typescript

  const tmp = value?.split(',').map((item: any) => item ? Number(item) : '');
```

### 2.2 树形数据操作引擎 ⭐⭐⭐

**位置**: `src/pages/TCList/TCModel/components/edit.ts` (209行)

#### 难点分析

事件树是一个**可编辑的嵌套结构**，需要支持：
1. **递归增删改查**: 任意层级节点的CRUD操作
2. **Key唯一性保证**: 复制/添加时重新生成所有子孙节点的Key
3. **操作权限控制**: 不同类型节点（select/map）有不同的操作权限
4. **递归校验**: 遍历所有节点收集校验失败的Key

#### 核心算法

**Key重新生成算法**:
```typescript
export const updateKeysForAddOrCopy = (
  data: treeItem[], 
  type: 'ADD' | 'COPY', 
  parentKey = ''
) => {
  data?.forEach((item: treeItem) => {
    if (item?.key) {
      // 保留原始Key的最后一段作为标识
      const currentKeys = item?.key?.split('-');
      item.key = newKey + currentKeys.pop();
      
      // ADD模式清空值，COPY模式保留值
      if (type === 'ADD') delete item?.value;
    }
    
    // 子节点使用随机数生成唯一Key
    if (item?.children && Array.isArray(item?.children)) {
      newKey = parentKey 
        ? `${parentKey}-${(Math.random() * 1000000).toFixed(0)}` 
        : item?.key;
      updateKeysForAddOrCopy(item?.children, type, newKey);
    }
  });
  return [{ ...data[0], key: parentKey }];
};
```

**操作权限Map生成**:
```typescript
export const generateItemToKey = (data: treeItem[], map = new Map()) => {
  for (const item of data) {
    if (item.type === 'select') {
      // select类型: 子节点可复制，多于1个时可删除
      item.children?.forEach((ch) => map.set(ch.key + 'copy', true));
      if (item.children?.length > 1) {
        item.children.forEach((ch) => map.set(ch.key + 'delete', true));
      }
    } else if (item.type === 'map') {
      // map类型: 子节点可复制、可删除、可添加map
      item.children?.forEach((ch) => {
        map.set(ch.key + 'map', true);
        map.set(ch.key + 'copy', true);
      });
      if (item.children?.length > 1) {
        item.children.forEach((ch) => map.set(ch.key + 'delete', true));
      }
    }
    // 递归处理子节点
    if (item?.children) generateItemToKey(item?.children, map);
  }
  return map;
};
```

**递归校验算法**:
```typescript
export const validateAllKeys = (data: treeItem[]) => {
  const typeLists = ['text', 'float', 'select', 'digit', 'bool', 
                     'digitArray', 'floatArray', 'textArray'];
  let keys: string[] = [];
  
  data?.forEach((item) => {
    if (!item?.children && item.require && typeLists.includes(item?.type)) {
      // 校验空值
      if (!isNotEmpty(item?.value)) {
        keys.push(item?.key);
      } else {
        // 校验格式规则
        if (item?.type === 'digitArray') {
          const expr = /^\d+(?:(,\d+)+)?$/.test(item?.value);
          if (!expr) keys.push(item?.key);
        } else if (item?.type === 'floatArray') {
          const expr = /^(-?\d+(\.\d+)?)(,-?\d+(\.\d+)?)*$/.test(item?.value);
          if (!expr) keys.push(item?.key);
        }
      }
    }
```

### 2.3 SSE实时日志流 ⭐⭐

**位置**: `src/pages/FgcgenList/index.tsx`

#### 难点分析

需要实现**实时日志流**，类似`tail -f`的效果。传统方案对比：

| 方案 | 延迟 | 资源消耗 | 实现复杂度 |
|------|------|----------|------------|
| 轮询（setInterval） | 高（取决于间隔） | 高（重复请求） | 低 |
| ReadableStream | 低 | 低 | 高（递归读取+AbortController） |
| **SSE（EventSource）** | **低** | **低** | **低（浏览器原生API）** |

#### 技术方案：SSE（Server-Sent Events）

```typescript
useEffect(() => {
  if (!logModalOpen || !fgcgen) return;

  // 1. 创建EventSource连接（浏览器原生SSE客户端）
  const eventSource = new EventSource(
    `${getApiPrefix()}/Fgcgen/logs?namespace=${fgcgen.namespace}&pod=${fgcgen.name}`,
  );

  // 2. 监听服务端推送的消息
  eventSource.onmessage = (event) => {
    setLogData((prev) => prev + event.data);
  };

  // 3. 连接错误时仅记录日志，让浏览器自动重连
  eventSource.onerror = () => {
    console.error('SSE error, will auto-reconnect');
  };

  // 4. 组件卸载或依赖变化时清理
  return () => {
    eventSource.close();
    setLogData('');
  };
}, [logModalOpen, fgcgen]);
```

#### 对比原ReadableStream方案

| 维度 | ReadableStream（旧） | SSE / EventSource（新） |
|------|---------------------|------------------------|
| 代码量 | ~48行 | ~20行 |
| 手动递归 | 需要`processChunks`递归读取 | 无需，浏览器自动处理 |
| 取消机制 | `AbortController`手动管理 | `eventSource.close()`一行搞定 |
| 断线重连 | 需自行实现 | 浏览器内置自动重连 |
| 后端要求 | `text/plain; chunks=true` | `text/event-stream` |
| 数据格式 | 原始二进制chunk | `data: ...

` 文本协议 |

#### 关键点

- **EventSource**是浏览器原生SSE客户端，专门用于接收服务端推送
- 服务端返回`Content-Type: text/event-stream`，每条消息以`data: ...

`格式发送
- `onmessage`回调自动处理消息解析，无需手动解码
- 组件卸载时`close()`断开连接，防止内存泄漏
- 后端需配合将日志接口改为SSE协议输出

```typescript
  type: 'bool',
  render: (record, mode) => (

        min: Number.MIN_SAFE_INTEGER,

const strategyMap = new Map<string, RenderStrategy>();
strategyMap.set('bool', BoolStrategy);

strategyMap.set('text', TextStrategy);

strategyMap.set('select', SelectStrategy);
strategyMap.set('digitArray', DigitArrayStrategy);
```

```typescript
  buildPathMapping(): this {
    ['amf', 'udm', 'pcf', 'chf', 'nrf', 'upf', 'sgw'].forEach((nf) => {
      this.traverseNFConfig(this.formConfiguration[nf]);
    });
    return this;
  }

  // 步骤3：递归遍历
  private traverseNFConfig(obj: any): void {
    if (Array.isArray(obj)) {
      obj.forEach((item) => this.traverseNFConfig(item));
    } else if (typeof obj === 'object' && obj !== null) {
      if (obj?.controlType === 'form' && obj?.path !== '') {
        Object.keys(obj.data || {}).forEach((key) => {
          this.pathMapToData.push(`${obj.path}.${key}`);
        });
      }
      Object.keys(obj).forEach((key) => {
        if (key === 'children' && Array.isArray(obj[key])) {
          this.traverseNFConfig(obj[key]);
        }
      });
    }
  }

  // 构建结果
  build() {
    return {
      formConfiguration: this.formConfiguration,
      pathMapToData: this.pathMapToData,
    };
  }
}

// 使用
const result = new FormConfigBuilder()
  .buildNFMapping(formJson)
  .buildPathMapping()
  .build();
```

### 13.1 React并发模式

#### Q11: React 19的并发模式是什么？项目中如何应用？

**答**:

**并发模式核心**:
- React可以同时准备多个版本的UI
- 高优先级更新（用户输入）可以打断低优先级更新（数据加载）
- 通过`startTransition`和`useDeferredValue`标记低优先级更新

**本项目应用**:

```typescript
// 1. startTransition - 标记非紧急更新
const handleSave = (record: treeItem, value: any) => {
  // 先同步更新ref（紧急）
  eventRef.current = eventRef.current?.map((item) =>
    item.key === data.key ? { ...data, tree: recordList } : item
  );
  // 再异步更新state（非紧急）
  startTransition(() => setRecordList(recordList));
};

// 2. useDeferredValue - 延迟渲染
const tree = useMemo(() => recordList, [JSON.stringify(recordList)]);
const deferredQuery = useDeferredValue(tree);
// 使用deferredQuery渲染表格，优先级低于用户输入
<EditableProTable value={deferredQuery} />
```

**优势**:
- 编辑表格时不会卡顿
- 模式切换更流畅
- 大数据量更新不阻塞交互

#### Q12: useMemo和useCallback的区别？什么时候用哪个？

**答**:

|------|------|----------|------|
| `useMemo` | 缓存计算结果 | 返回值 | 计算依赖 |

```typescript
  // ...
```
