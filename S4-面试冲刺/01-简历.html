实时通信

  • WebSocket — 背压控制、消息合并、二进制协议编码、心跳保活、断线重连、指数退避 + jitter

  • Web Worker — Worker Pool 管理、自适应分区、多路归并、生产/消费模式

  • React 19 编译器 — 构建期自动 memo，零运行时开销

工程化

  • CI/CD — GitLab CI 流水线（validate → build → package → deploy）
  • 部署 — Docker 多阶段构建 + Helm Chart + K8s 滚动更新

可视化 & 后端

  • 可视化 — ECharts 6、OpenLayers 10.9、Grafana、Prometheus
  • Go 后端 — Gin + Gorilla WebSocket + golang-jwt + 分片上传、文件合并、SHA-256 校验
  • 状态管理 — Zustand 5（按 selector 精确订阅、persist 持久化）

AI 工具链 — Open Code、Trae、Codex 辅助开发与 Prompt Engineering

项目一：5G核心网测试用例管理系统 ​

技术栈： React 19 + TypeScript 6 + Ant Design 6 + Zustand 5 + ECharts + SSE + Go + Gin

背景：在5G核心网的开发与测试过程中，测试工程师需要对大量的网络功能（NF）进行配置和验证，传统 UI 编码方式无法支撑测试用例频繁变动，每次测试场景变更需开发介入改代码发版，严重制约测试效率。本项目旨在构建一个可视化的测试用例管理系统，实现对5G核心网测试环境的统一管理和自动化测试。

  • S（情境）：测试用例频繁变更，每次场景变更需开发改代码发版，严重制约测试效率
  • T（任务）：构建可视化测试用例管理系统，实现非前端人员零代码配置测试场景

我的行动与成果：

  • 递归动态表单引擎：自研 JSON Schema 渲染引擎，Schema 抽象为四层 AST 树（tabs → card → form → leaf），递归渲染器逐层解析，策略模式注册字段组件；支持条件显隐表达式、字段联动自动填充、四级校验体系（同步/异步/AJV/后端）、实时 JSON 编辑双向绑定 → 8 种字段类型，新增仅需一行 registerField()，人效提升 80%
  • 树形数据操作引擎：自研递归 CRUD 操作库，支持任意层级节点增删改查、dnd-kit 拖拽排序、Key 唯一性保证算法、操作权限 Map 生成与递归校验 → 支撑复杂测试用例可视化编排
  • 可编辑树表格性能优化：应用 useDeferredValue + startTransition，500ms 防抖校验 + DOM 错误高亮 → 编辑性能提升 40%
  • SSE 实时日志流：利用 SSE + ReadableStream 构建实时日志流式传输，AbortController 控制连接生命周期，正则算法实现异常状态秒级高亮 → 问题定位效率提升 50%
  • 大文件断点续传：SHA-256 分片校验 + Zustand persist 持久化 + 滑动窗口并发 + 指数退避重试 + Promise Park 暂停/恢复 + 刷新后续传对账 → 500MB 文件 UI 零阻塞
  • K8s 云原生集成：实现 Kubernetes YAML 在线编辑与下发 → 支撑日均 200+ 自动化测试任务

项目二：AeMS — 企业级综合网络管理系统 ​

技术栈： React 19 + TypeScript 6 + Ant Design 6 + Zustand 5 + WebSocket + OpenLayers 10.9 + ECharts 6 + Go + Gin

背景：面向数十万台 Small Cell 网元设备的综合智能网管中枢，覆盖设备监控、智能告警、日志分析、安全审计全链路，需处理 1000+ QPS 告警并发与 十万级设备 GIS 拓扑渲染。

  • S（情境）：数十万台设备实时监控，原系统在万级并发下卡顿严重
  • T（任务）：支持 1000+ QPS 高并发实时监控 + 十万级 GIS 拓扑渲染

我的行动与成果：

  • GIS 十万级点位四重优化：BBOX 视口裁剪仅保留可见点位 → Cluster 聚类聚合（distance=40px）→ dataCache 全量缓存（zoom+extent 作为 key）→ moveend 惰性刷新 + RAF 节流 → 渲染量降至百级，帧率 <10fps → 60fps
  • 多协议降级传输层：统一 Transport 接口抽象，三级降级链（WebSocket → SSE → Polling）；背压控制（bufferedAmount > 1MB 排队）+ 消息合并（16ms/64KB 批量发送）+ 二进制协议编码（payload ↓30%+）+ 心跳保活（30s Ping / 10s Pong）+ RAF 双缓冲渲染 → 4000 msg/s 全帧率 60fps
  • LRU 路由缓存：display:none 保持页面状态而非卸载，访问计数淘汰 + 写后失效缓存一致性（staleKeys 标记过期页面，切换时自动刷新） → 页面切换性能提升 60%
  • RBAC 位编码权限：位运算 O(1) 权限检查（6 种权限编码在 1 个 number 中，仅 4 字节），三层联动控制（菜单递归过滤 / 路由 Guard 拦截 / 按钮 ACL） → 权限越权漏洞降低 90%
  • 双 Token 无感刷新：Promise gate 并发请求排队 + Token Rotation 防重放 + Replay 检测 → 平台可用性达 99.9%，客诉量下降 40%
  • 请求加载 Signal：方法-路径匹配树（method:path 作为 key），拦截器自动标记，按钮组件通过 loadingMap 自动关联 → 消除按钮 Loading 等待

项目三：网元运维与数据管理系统 ​

技术栈： React 19 + TypeScript 6 + Ant Design 6 + Zustand 5 + Go + Prometheus + Grafana

背景：5G 核心网元（SMF / UPF）的运维与数据管理系统，覆盖网元全生命周期：注册接入、状态监控、配置下发、日志采集解密、备份恢复、审计溯源。

  • S（情境）：一线运维排查故障需预览日志文件，但 25MB 日志直接让浏览器 OOM 崩溃
  • T（任务）：浏览器端无痛预览百万行级日志，保持流畅的搜索和滚动体验

我的行动与成果：

  • 生产/消费模式流式解密：主线程 fetch 分片读取（ReadableStream）→ postMessage 发送给 Worker → Worker 并行 XOR/RSA 解密 → postMessage 发回主线程逐块追加；Worker Pool 根据 CPU 核心数动态创建 → 首段流式输出，UI 零卡顿
  • 虚拟滚动：固定行高 + 可视区域裁剪 + O(1) 滚动位置计算，仅渲染可视区域 30-50 行 → 支持千万级数据量流畅展示
  • Web Worker 分治归并：Worker Pool（navigator.hardwareConcurrency）+ 自适应分区 + 多路归并算法 → 100 万数字排序 620ms → 180ms（3.4×），主线程零阻塞
  • 声明式表单框架：注册表 + 工厂模式设计，registerField() 一行注册新字段 → 支持 9 种控件类型与 5 个跨字段验证器
  • 全链路可观测体系：深度集成 Prometheus 采集指标，内嵌 Grafana 构建多维可视化监控看板 → 实现网元健康度与核心性能的秒级监控与趋势预警

- 背压控制 + 消息合并，防止内存溢出
- 二进制协议编码，减少 payload 体积 30%+
- RAF 双缓冲渲染，保持 60fps 流畅度

同时还有 SSE 日志流（ReadableStream + AbortController）
和双 Token 无感刷新（Promise gate + Token Rotation），
构成完整的实时通信能力矩阵。

━━━ 第二，性能优化实践 ━━━

我重点攻克了四个性能瓶颈：

1. Web Worker 分治归并排序与日志解密：
   利用 Worker Pool + 自适应分区，将排序和解密计算转移到独立线程，
   不阻塞主线程 UI 渲染。生产/消费模式实现流式输出。

2. GIS 十万级点位渲染：
   四重优化（Cluster 聚合 → BBOX 裁剪 → dataCache 缓存 → 惰性刷新），
   帧率从 <10fps 提升到 60fps。

3. LRU 路由缓存：
   用 display:none 保持页面状态，结合写后失效机制保证缓存一致性，
   限制最多 5 个缓存页面防内存溢出。

4. 百万行日志流式解密：
   生产/消费模式 + Web Worker XOR 解密 + 虚拟滚动，
   支持千万级数据量流畅展示。
