# 数组 push vs unshift 性能测试

## 简介

对比数组 `push`（尾部添加）和 `unshift`（头部添加）在大量数据下的性能差异。`unshift` 每次操作需要将数组所有元素后移，时间复杂度 O(n)，而 `push` 是 O(1)，因此大数据量下 `unshift` 远慢于 `push`。

**结论**：在需要频繁头部操作时，应使用链表或倒序处理代替数组 `unshift`。

## 代码实现

```javascript
/**
 * 题目：数组 push vs unshift 性能测试
 * 描述：对比 push（尾部添加）和 unshift（头部添加）在大量数据下的性能差异。
 *       unshift 每次操作需要将数组所有元素后移，时间复杂度 O(n)，
 *       push 是 O(1)，因此大数据量下 unshift 远慢于 push。
 * 结论：在需要频繁头部操作时，应使用链表或倒序处理代替数组 unshift。
 */
const arr = [];
console.time('perfTest');
for (let i = 0; i < 100000; i++) {
  arr.unshift(i); // 每次插入都要移动所有元素，性能极差
}
console.timeEnd('perfTest');
```

## 逐行解析

| 行号 | 代码 | 说明 |
|------|------|------|
| 8 | `const arr = []` | 创建一个空数组 |
| 9 | `console.time('perfTest')` | 启动计时器，标签为 `perfTest` |
| 10-12 | `for` 循环 | 循环 10 万次，每次在数组头部插入 `i` |
| 11 | `arr.unshift(i)` | 在数组头部插入元素，内部所有已有元素需要依次后移一位，时间复杂度 O(n) |
| 13 | `console.timeEnd('perfTest')` | 停止计时器并打印耗时 |

## 复杂度分析

| 操作 | 时间复杂度 | 原因 |
|------|-----------|------|
| `push` | O(1) | 直接在尾部追加，无需移动元素 |
| `unshift` | O(n) | 插入头部后，所有元素必须后移一个位置 |

## 运行结果参考

执行上述代码（10 万次 `unshift`），在一般机器上耗时约 **数秒到十几秒**，而同样的 `push` 操作只需几毫秒。这个差距会随着数据量的增加急剧扩大。
