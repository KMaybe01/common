## 🚀 阶段六：Go 新特性与前沿技术 (2023-2026)

### 6.1 泛型 (Generics)
#### Q31: Go 1.18 引入的泛型解决了什么问题？使用场景有哪些？

**难度**：⭐⭐ | **频率**：📌 常考

**考点**：泛型约束、代码复用、性能优化。

**💡 记忆关键词**：消除重复、约束 Constraints、数据结构、避免 interface{}

**答案要点**：
- **解决问题**：消除重复代码（如针对 `int`, `int64`, `float64` 写多个相同逻辑的函数），避免使用 `interface{}` 带来的运行时类型断言开销。
- **约束 (Constraints)**：使用 `comparable`、`any` 或自定义 interface 约束类型参数。
- **场景**：通用数据结构（链表、树、Set）、算法库（排序、查找）、工具函数（指针获取 `Ptr[T](v T) *T`）。
- **注意**：泛型不适合替代接口进行多态设计，也不适合仅针对单一类型的方法。


### 6.2 语言细节更新
#### Q32: Go 1.22 对 `for` 循环变量作用域做了什么重大修复？

**难度**：⭐⭐ | **频率**：📌 常考

**考点**：闭包陷阱、语义变更、向后兼容。

**💡 记忆关键词**：闭包陷阱、每次迭代新变量、Go1.22 修复、向后兼容

**答案要点**：
- **旧行为**：`for` 循环变量在所有迭代中共享同一个内存地址，导致在 `goroutine` 或闭包中引用时，最终获取的都是最后一次迭代的值。
- **新行为 (Go 1.22+)**：每次迭代创建新的循环变量，闭包捕获的是当次迭代的值，彻底修复了经典的"闭包陷阱"。
- **影响**：代码更安全，但需注意依赖旧行为的极端边界情况（极少见）。


#### Q33: Go 1.21+ 引入的 `min`/`max`/`clear` 内置函数有什么作用？

**难度**：⭐ | **频率**：📖 了解

**考点**：内置函数增强、代码简化。

**💡 记忆关键词**：min/max 内置、clear 清空、有序类型、复用内存

**答案要点**：
- `min(a, b...)` / `max(a, b...)`：支持任意有序类型，无需再写三目运算符或引入 `math` 包。
- `clear(m)`：清空 Map 或 Slice，底层复用内存，比重新 `make` 更高效。


#### Q34 (新增): Go 1.21 引入的 `slices`、`maps` 和 `cmp` 标准库包有什么作用？

**难度**：⭐⭐ | **频率**：📌 常考

**考点**：泛型标准库、`slices` 操作、`maps` 操作、比较器。

**💡 记忆关键词**：slices 泛型操作、maps 泛型操作、cmp.Compare、声明即用

**答案要点**：
- **`slices` 包**：提供泛型切片操作函数，如 `slices.Sort`、`slices.Index`、`slices.Delete`、`slices.Clip`、`slices.Compact`、`slices.Contains`。消除了手动编写切片工具函数的需要。
- **`maps` 包**：提供泛型 Map 操作函数，如 `maps.Clone`、`maps.DeleteFunc`、`maps.Copy`。
- **`cmp` 包**：提供有序类型的比较函数 `cmp.Compare[T constraints.Ordered](a, b T) int` 和 `cmp.Less`、`cmp.Or`。
- **优势**：类型安全，编译期检查，无需 `interface{}` 断言。

```go
import (
    "cmp"
    "maps"
    "slices"
)

// 排序任意可比较切片
names := []string{"c", "a", "b"}
slices.Sort(names)

// 从切片删除元素
names = slices.Delete(names, 0, 1)

// 克隆 Map
src := map[string]int{"a": 1, "b": 2}
dst := maps.Clone(src)

// 比较器
result := cmp.Compare(5, 10) // -1
```


#### Q35 (新增): Go 1.22 增强的 `http.ServeMux` 路由有哪些改进？

**难度**：⭐⭐ | **频率**：📌 常考

**考点**：方法匹配、路径参数、通配符。

**💡 记忆关键词**：HTTP 方法前缀、通配符 `{name}`、`{*}path`、标准库增强

**答案要点**：
- **方法匹配**：支持 `GET /path`、`POST /path` 等前缀方法模式，无需第三方路由库。
- **路径参数**：使用 `{name}` 通配符捕获路径段，通过 `r.PathValue("name")` 获取。
- **通配符后缀**：`{*}path` 匹配剩余路径段。
- **默认模式**：`GET /path` 匹配 GET 请求，`/path` 匹配所有方法。

```go
mux := http.NewServeMux()
mux.HandleFunc("GET /api/users/{id}", func(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")
    fmt.Fprintf(w, "User ID: %s", id)
})
mux.HandleFunc("POST /api/users", createUser)
mux.HandleFunc("GET /api/static/{*}path", serveStatic)

// Go 1.22 前的写法：
// mux.HandleFunc("/api/users/", ...)
// 需手动解析路径
```


#### Q36 (新增): Go 1.21 的 `log/slog` 结构化日志库是什么？

**难度**：⭐⭐ | **频率**：📌 常考

**考点**：结构化日志、Logger、Handler、性能。

**💡 记忆关键词**：Key-Value 对、JSON/Text Handler、零依赖、`slog` 替代 `log`

**答案要点**：
- `log/slog` 是 Go 1.21 引入的标准库结构化日志，无需外部依赖。
- **三大组件**：`Logger`（日志记录器）、`Record`（日志记录）、`Handler`（输出处理）。
- **内置 Handler**：`slog.NewJSONHandler`（JSON 格式）和 `slog.NewTextHandler`（Key=Value 格式）。
- **优势**：统一结构化日志标准，支持等级控制、函数级参数分组、上下文传递。

```go
import "log/slog"

// JSON 日志
logger := slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
    Level: slog.LevelInfo,
}))

logger.Info("user login", "user_id", 123, "ip", "192.168.1.1")
logger.Error("db timeout", "err", err, "query", "SELECT ...")
```


### 6.3 Go 1.23-1.25 最新特性

#### Q37 (新增): Go 1.23 的迭代器（Iterators）和 `range-over-func` 是什么？

**难度**：⭐⭐⭐ | **频率**：📌 常考

**考点**：迭代器模式、`range` 扩展、标准库适配。

**💡 记忆关键词**：range-over-func、迭代器契约、Pull/Push 模式、标准库迭代器

**答案要点**：
- **`range-over-func`**：Go 1.23 支持 `range` 遍历自定义函数迭代器，允许用户类型实现 `range` 协议。
- **迭代器类型**：`func(yield func(T) bool)` 或 `func(yield func(K, V) bool)`。
- **标准库迭代器**：Go 1.23 在 `slices`、`maps` 包中增加了迭代器支持，如 `slices.All`、`slices.Values`、`slices.Backward`、`maps.All`、`maps.Keys`。
- **Pull/Push API**：提供 `iter.Pull` 和 `iter.Pull2` 将 Push 迭代器转为 Pull 风格。

```go
// 自定义迭代器
func Backward[E any](s []E) func(func(int, E) bool) {
    return func(yield func(int, E) bool) {
        for i := len(s)-1; i >= 0; i-- {
            if !yield(i, s[i]) {
                return
            }
        }
    }
}

s := []string{"a", "b", "c"}
for i, v := range Backward(s) {
    fmt.Println(i, v)
}

// 标准库迭代器
for i, v := range slices.Backward(s) {
    fmt.Println(i, v)
}
for k, v := range maps.All(m) {
    fmt.Println(k, v)
}
```
